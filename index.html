Case Study

From friction to flow.

A deep-dive into how a healthcare platform went from a high-bounce rate to a high-trust patient experience.

Healthcare · Web Application

MediCare Patient Portal

Challenge

A website that made patients anxious instead of confident

The existing portal had a 78% bounce rate on the appointment booking flow. Confusing navigation, slow load times, and a visual design that hadn't been touched since 2018 were actively costing the clinic new patients. The goal: transform a liability into a competitive advantage.

Design Thinking

Empathy mapping revealed the real friction

Through user interviews and session recordings, we identified that patients were abandoning primarily because of trust signals — not the booking form itself. We redesigned around confidence: clear credentials, visible social proof, transparent pricing, and a booking flow reduced from 7 steps to 3.

Development

Performance-first, built on modern infrastructure

Built with Next.js and Supabase for real-time appointment availability. Deployed on Netlify's edge network for sub-100ms global load times. Image optimization, lazy loading, and code splitting kept the bundle lean — achieving a Lighthouse performance score that consistently exceeds 95.
